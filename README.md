# Portfolio Site 👋

I decided to recreate my personal site using multiple frameworks:
* [SvelteKit](https://kit.svelte.dev/) as meta framework for combining Single Page Application capabilities with Server Side Rendered content
* [Svelte](https://svelte.dev/) as UI development paradigm for web applications with a small footprint
* [TailwindCSS](https://tailwindcss.com) a utility-first CSS framework
* [DaisyUI](https://daisyui.com) an open source Tailwindcss component library

Plugins I've added
* [MDsveX](https://www.npmjs.com/package/mdsvex) is a MarkDown preprocessor for Svelte, which enables .md and .svelte to be interpreted within the same files
* [TailwindCSS Typography](https://tailwindcss.com/docs/typography-plugin) introduces default font facings and styling for text with an additional "prose"-class 

Languages used
* **TypeScript** since I am not a fan of the loosely typed nature of JavaScript. Since SvelteKit has TypeScript support, I decided to use TypeScript instead of JavaScript
* **CSS** to tweak styling and create custom components that aren't part of DaisyUI
* **HTML** to create custom components
* **MarkDown** since I am a huge fan of the diverse utility of this markup languange and have written my previous content with this language

## 📚 Learning goal
The main star of the project is SvelteKit. This meta framework handles the core capabilities of the site and greatly eases the development pains since it handles the non-functional requirements out of the box.

In combination with Tailwindcss and using a component library such as DaisyUI, a lot of the actual UI design is based on proven concepts. This greatly reduces the time it takes to design, develop and test components.

Instead I can put my time in focusing on delivering value to the end user.

**Redesign my whole site while learning a JavaScript framework**
